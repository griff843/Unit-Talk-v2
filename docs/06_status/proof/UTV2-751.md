### 3. Null bookmaker_key Diagnosis

~24% of all `provider_offers` rows have `bookmaker_key = NULL`. Sample market keys from null-book rows:
- `points-all-game-ml` (moneyline)
- `points-all-game-ou` (total)
- `points-all-game-sp` (spread)
- `points-all-1h-ml`, `points-all-2q-sp`, etc.

**Verdict: EXPECTED — SGO consensus/fair-line aggregate rows.** These are the `fairOdds` and `bookOdds` fields from SGO, which represent vig-free and vig-included consensus across all available books. They do not carry a per-bookmaker key by design. Per `PROVIDER_KNOWLEDGE_BASE.md` §3.7, consensus fields have an approved use set (operator reference, intelligence overlays) — they are not used for per-book CLV computation. This is **not a bug**.

The proof script and backfill runner are ready when needed:

# Run backfill for MLB (2026-03-28 to 2026-04-19)
npx tsx scripts/proof/utv2-751-sgo-historical-backfill.ts --backfill --sport MLB

The backfill uses the existing `runHistoricalBackfill()` infrastructure (`apps/ingestor/src/historical-backfill.ts`), which iterates day-by-day and uses SGO's date-range filtering. No new infrastructure required.
