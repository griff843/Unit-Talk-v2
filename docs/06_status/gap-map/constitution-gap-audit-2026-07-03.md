# Constitution Gap Audit — 2026-07-03

**ADVISORY — not binding governance.** This report is Fable-5-run analysis of the Unit Talk V2 constitution/governance system. It does not modify any file, weaken PM authority, or take effect on its own. Findings become binding only through PM-ratified child issues and governance PRs (see Linear UTV2-1414..1430 for the Wave 1 tracking structure).

## Executive verdict: PARTIAL

The governance system is real, not theater: the T1 `t1-approved` label is mechanically enforced by `merge-gate.yml`, the Phase 7A brake is enforced in code and at deploy time, delivery is fail-closed, settlement records are append-only, and no unguarded path to public/member-facing delivery was found.

It is not a PASS for three reasons. First, the audit trail itself is compromised: `ops:lane-close --repair-merged` writes synthetic `verdict:"pass"` entries into `truth_check_history` that no truth-check produced. Second, merge authority for T2 is defined five different, mutually contradictory ways across the canon, and PM approval itself is defined three incompatible ways (label vs PR comment vs in-session chat) — with chat-based gates directly contradicting `EXECUTION_TRUTH_MODEL.md`'s rank-5 "chat is never authoritative" rule. Third, several written gates are not actually enforced (R-levels are not read by truth-check and their CI check is not a required branch-protection context), while one enforced gate demands the impossible (artifacts from a gitignored directory).

5. **G-11 + G-12 — launch safety floor.** Incident runbook, rollback procedure, delivery kill switch, default-disabled public targets, mandatory alerting.

## Over-governing vs under-governing

**Over-governed (low-risk work):** A one-line T3 doc edit runs the full machine — live governor scripts, three-brain routing, isolated worktree with `pnpm install`, R-level check, full `pnpm verify`, manifest + sync + lease + merge mutex + truth-check. `three-brain.md:143` already admits "overhead exceeds value" for Codex T3 but Claude T3 pays it anyway.

**Under-governed (high-risk work):** model quality (edge floor 0, calibration unwired, acceptance standard unratified), delivery blast radius (no kill switch, default-enabled public targets), the repair/bypass surfaces (`--repair-merged`, `--singleton-approved`, admin-merge pass-through, self-declared file scope), and DB disaster recovery. Process risk is over-governed; product and runtime risk is under-governed.
