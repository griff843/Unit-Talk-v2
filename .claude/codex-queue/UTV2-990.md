# UTV2-990: Replace GHA-only ingestor with persistent supervised production process

**Branch:** `codex/utv2-990-persistent-ingestor-supervised-service`
**Tier:** T2
**File scope lock:** `apps/ingestor/src/**`, `scripts/ingestor-supervisor.ts`, `scripts/ops/ingestor-health-check.ts`, `docker-compose.yml`, `docker-compose.prod.yml`

## Context

The ingestor is currently configured as a Docker Compose service (`restart: always` in `docker-compose.prod.yml`) but in practice GitHub Actions cron is the primary ingestion mechanism. The Claude + Codex audit found the ingestor supervisor DOWN and provider offers stale.

Key facts:
- `scripts/ingestor-supervisor.ts` manages a local Node process (dev/local mode)
- `docker-compose.prod.yml` already has `ingestor` service with `restart: always` — this is the right prod mechanism
- `pnpm ingestor:status` calls `ingestor-supervisor.ts status` — checks local process state, not container state
- GHA scheduled workflow triggers the supervisor script directly (acts as primary, not backup)
- `pnpm stage:freshness` runs `scripts/stage-freshness-checks.ts`

## What to implement

### 1. Add Docker container health check to `pnpm ingestor:status`

Update `scripts/ingestor-supervisor.ts` `status` command to additionally check if the Docker container `unit-talk-ingestor-1` (or equivalent) is running. Output should show:
- Container mode (Docker): running/stopped + last restart time
- Local process mode: existing behavior
- If `UNIT_TALK_INGESTOR_RUNTIME_MODE=fail_closed` is set, warn if neither container nor local process is running

### 2. Make GHA the backup, not primary

Check `scripts/ingestor-supervisor.ts` `start` command. Add a guard: if the ingestor Docker container is already running and healthy (per `docker inspect` or `docker ps`), skip the local supervisor spawn and log "Container mode active — local supervisor not needed". This prevents GHA from spawning a duplicate process.

### 3. Add `scripts/ops/ingestor-health-check.ts`

New script that:
- Checks Docker container status via `docker ps --filter name=ingestor`
- Checks latest `provider_offers` record freshness from Supabase (use `@unit-talk/config` + `@supabase/supabase-js`)
- Outputs JSON: `{ healthy: boolean, containerRunning: boolean, offerAgeMinutes: number, staleSince?: string }`
- Used by `pnpm runtime:health` and CI

Add `"ingestor:health-check": "tsx scripts/ops/ingestor-health-check.ts"` to root `package.json`.

### 4. Update `pnpm ingestor:alert-check` (scripts/ingestor-alert-check.ts)

Ensure it calls the new `ingestor-health-check.ts` logic and exits non-zero if `healthy: false`.

## Acceptance criteria

- `pnpm ingestor:status` shows container health when Docker is running
- `pnpm ingestor:health-check` exits 0 when container is up AND offers are fresh (<30 min)
- `pnpm ingestor:start` skips local spawn if Docker container is already running
- `pnpm ingestor:alert-check` exits non-zero when stale
- `pnpm verify` exits 0

## Do NOT

- Touch DB migrations or `database.types.ts`
- Modify `apps/ingestor/src/` business logic — only scripts and health checks
- Modify `docker-compose.prod.yml` image tags or resource limits
- If any T1/Tier-C path is needed, STOP and report back
