## Task packet

### Linear issue
UTV2-497 — **P7B-03 - Correct ingress contract and runtime flag semantics**

### Why it matters
During Phase 7 ratification (UTV2-485, 2026-04-10), PM re-grounded the ingress model: autonomous non-human producers (`system-pick-scanner`, `alert-agent`, `model-driven`) are **transitional and deprecated**, not a long-term design. They currently submit directly to `/api/submissions` because the governed upstream adapters (P7B-02a, P7B-02b) haven't landed yet — but docs and contract-adjacent language from earlier phases still treat direct non-human submission as a permanent architectural choice.

This task corrects that language so future readers (and future agents) don't re-entrench the transitional pattern as design truth.

This is **not** a runtime change. It is pure language correction in docs and contract file comments. No types move, no runtime code touches, no migrations.
